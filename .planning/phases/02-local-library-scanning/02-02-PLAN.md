    2. Create `internal/cli/scan.go`:
       - Package `cli`
       - Import: scanner, metadata, db, config packages; fmt, os, time
       - `var scanRecursive bool`
       - `var scanCmd = &cobra.Command{Use: "scan", Short: "Scan local library for audiobooks", Long: "...", RunE: runScan}`
       - `func init()`: add `--recursive/-r` flag bound to scanRecursive (default from viper scan.recursive). Register scanCmd with rootCmd via `rootCmd.AddCommand(scanCmd)`.
       - `func runScan(cmd *cobra.Command, args []string) error`:
         a. Get library_path from viper. If empty, return error: `fmt.Errorf("library path not configured\n\nRun 'earworm config set library_path /path/to/library' to set your library location")`
         b. Validate library path exists and is a directory. If not: return `fmt.Errorf("library path %q does not exist or is not a directory\n\nCheck that your library path is correct and accessible: earworm config show", libPath)`
         c. Open database via `db.Open(config.DBPath())`. On error: return `fmt.Errorf("failed to open database: %w\n\nTry removing %s and running again", err, config.DBPath())`
         d. Defer database.Close().
         e. Print scan start message to stderr (not stdout, so --json piping works): `fmt.Fprintf(cmd.ErrOrStderr(), "Scanning %s...\n", libPath)`
         f. Call `scanner.ScanLibrary(libPath, scanRecursive)`. On error: return wrapped error with recovery hint.
         g. Print intermediate: `fmt.Fprintf(cmd.ErrOrStderr(), "Found %d books, extracting metadata...\n", len(discovered))`
         h. Call `scanner.IncrementalSync(database, discovered, metadata.ExtractMetadata)`.
         i. Print summary to stdout:
            ```
            Scan complete:
              Added:   %d
              Updated: %d
              Removed: %d
              Skipped: %d
            ```
         j. If len(skipped) > 0 and !quiet: print skipped folder summary to stderr: `Skipped %d folders:` then list first 10 with reason, and `... and %d more` if truncated.
       - Per D-10: The spinner with live counter is aspirational for NAS mounts. For this implementation, use simple stderr print statements showing progress at key milestones (scan start, books found, metadata extraction, complete). A goroutine-based spinner can be added as a polish step later if needed. This avoids pulling in bubbletea as a dependency.

    3. Create `internal/cli/status.go`:
       - Package `cli`
       - Import: db, config, encoding/json, fmt, os
       - `var jsonOutput bool`
       - `var statusCmd = &cobra.Command{Use: "status", Short: "Show library contents and status", RunE: runStatus}`
       - `func init()`: add `--json` flag. Register statusCmd with rootCmd.
       - `func runStatus(cmd *cobra.Command, args []string) error`:
         a. Open database via `db.Open(config.DBPath())`. On error: return `fmt.Errorf("failed to open database: %w\n\nRun 'earworm scan' first to initialize your library", err)`
         b. Defer database.Close().
         c. Call `db.ListBooks(database)`. On error: return wrapped error.
         d. If jsonOutput (D-07, LIB-06):
            ```go
            enc := json.NewEncoder(cmd.OutOrStdout())
            enc.SetIndent("", "  ")
            return enc.Encode(books)
            ```
         e. If len(books) == 0: print "No books in library. Run 'earworm scan' to index your library." and return nil.
         f. For styled output (D-06), print one-line-per-book format to cmd.OutOrStdout():
            ```
            Author - Title [ASIN] (status)
            ```
            Where status uses these indicators:
            - "scanned" -> OK
            - "downloaded" -> DL
            - "organized" -> OK
            - "error" -> ERR
            - "removed" -> GONE
            - "unknown" -> ?
            Add a total count at the end: `\n%d books total`
         g. Per D-08 (Claude's discretion): include `--author` filter flag (simple substring match on author field) and `--status` filter flag (exact match). These are lightweight and natural for CLI usage.

    4. Error handling strategy per CLI-03:
       - Every error returned from RunE should have format: `"what went wrong\n\nRecovery: how to fix it"`
       - The `\n\n` separator makes recovery hints visually distinct in terminal output.
       - Cobra's SilenceUsage=true (already set in root.go) prevents usage spam after errors.
