    1. Type definitions:
    ```go
    type IssueType string
    const (
        IssueNoASIN          IssueType = "no_asin"
        IssueNestedAudio     IssueType = "nested_audio"
        IssueMultiBook       IssueType = "multi_book"
        IssueMissingMetadata IssueType = "missing_metadata"
        IssueWrongStructure  IssueType = "wrong_structure"
        IssueOrphanFiles     IssueType = "orphan_files"
        IssueEmptyDir        IssueType = "empty_dir"
        IssueCoverMissing    IssueType = "cover_missing"
    )

    type Severity string
    const (
        SeverityError   Severity = "error"
        SeverityWarning Severity = "warning"
        SeverityInfo    Severity = "info"
    )

    type DetectedIssue struct {
        Path            string
        IssueType       IssueType
        Severity        Severity
        Message         string
        SuggestedAction string
    }
    ```

    2. Individual detector functions (each is a pure function):

    - `detectEmptyDir(dirPath string, entries []os.DirEntry) []DetectedIssue` — if len(entries)==0, return issue with SeverityWarning, message "Directory is empty", suggested "Delete empty directory"

    - `detectNoASIN(dirPath string, entries []os.DirEntry) []DetectedIssue` — check if any entry has .m4a/.m4b extension (case-insensitive). If audio found, call ExtractASIN(filepath.Base(dirPath)). If no ASIN, return issue with SeverityWarning, message "Audio files present but no ASIN in folder name", suggested "Add ASIN to folder name: rename to 'Title [ASIN]'"

    - `detectNestedAudio(dirPath string, entries []os.DirEntry) []DetectedIssue` — for each subdirectory entry, call metadata.FindAudioFiles(filepath.Join(dirPath, entry.Name())). If any subdir has audio, return issue with SeverityWarning, message including subdir name, suggested "Flatten: move audio files up to book directory"

    - `detectOrphanFiles(dirPath string, entries []os.DirEntry) []DetectedIssue` — known extensions: .m4a, .m4b, .m4a, .jpg, .jpeg, .png, .json, .nfo, .cue, .txt, .log, .xml. For non-dir entries with extension NOT in known set, return issue with SeverityInfo, message listing orphan filenames, suggested "Review and remove or relocate orphan files". Skip directories.

    - `detectCoverMissing(dirPath string, entries []os.DirEntry, meta *BookMetadata) []DetectedIssue` — check entries for .jpg/.jpeg/.png files. If none found AND (meta is nil OR meta.HasCover is false), AND there is at least one audio file in entries, return issue with SeverityInfo, message "No cover image found", suggested "Add cover image to book directory"

    - `detectMissingMetadata(dirPath string, entries []os.DirEntry, meta *BookMetadata) []DetectedIssue` — if meta is nil (extraction failed) AND no entry named "metadata.json" exists, AND there is at least one audio file, return issue with SeverityInfo, message "No metadata extractable from audio files", suggested "Write metadata.json sidecar with known information"

    - `detectWrongStructure(dirPath string, libraryRoot string) []DetectedIssue` — compute depth = count of filepath.Separator in strings.TrimPrefix(dirPath, libraryRoot). If depth > 2 AND directory contains audio files (check via entries or pass flag), return issue with SeverityInfo, message "Directory is nested too deeply", suggested "Restructure: move to Author/Title [ASIN] format". Note: this detector needs libraryRoot param — DetectIssues passes it through.

    - `detectMultiBook(dirPath string, entries []os.DirEntry) []DetectedIssue` — conservative heuristic: get all .m4a/.m4b files, extract base names stripping numeric prefixes/suffixes (like "01 - ", " Part 1"). If the remaining base names form 2+ distinct groups (simple: count unique prefixes after stripping numbers), AND there are 2+ audio files, return issue with SeverityWarning. Message: "Multiple distinct titles detected in single directory". Suggested: "Split: separate into individual book directories". IMPORTANT: if all files share the same title prefix, return nil (it is a multi-disc single book).

    3. Aggregator:
    ```go
    func DetectIssues(dirPath string, entries []os.DirEntry, meta *BookMetadata, libraryRoot string) []DetectedIssue {
        var issues []DetectedIssue
        issues = append(issues, detectEmptyDir(dirPath, entries)...)
        issues = append(issues, detectNoASIN(dirPath, entries)...)
        issues = append(issues, detectNestedAudio(dirPath, entries)...)
        issues = append(issues, detectOrphanFiles(dirPath, entries)...)
        issues = append(issues, detectCoverMissing(dirPath, entries, meta)...)
        issues = append(issues, detectMissingMetadata(dirPath, entries, meta)...)
        issues = append(issues, detectWrongStructure(dirPath, libraryRoot)...)
        issues = append(issues, detectMultiBook(dirPath, entries)...)
        return issues
    }
    ```

    Note: detectWrongStructure needs libraryRoot — add it as a parameter to DetectIssues. detectNestedAudio uses metadata.FindAudioFiles which reads the filesystem — this is the only detector that touches disk beyond the provided entries.
