    Create `internal/planengine/` directory and two files.

    **engine.go** — package `planengine`:

    1. Define `Executor` struct with field `DB *sql.DB`.

    2. Define `OpResult` struct with fields:
       - `OperationID int64`
       - `Success bool`
       - `SHA256 string`
       - `Error string`

    3. Implement `func (e *Executor) Apply(ctx context.Context, planID int64) ([]OpResult, error)`:
       - Get plan via `db.GetPlan(e.DB, planID)`. Return error if nil.
       - Validate plan status is "ready", "running", or "failed". Return `fmt.Errorf("cannot apply plan with status %q", plan.Status)` for others.
       - If plan status is "running" or "failed", call `prepareResume()` to reset any "running" ops to "pending".
       - Set plan status to "running" via `db.UpdatePlanStatusAudited(e.DB, planID, "running")`.
       - List operations via `db.ListOperations(e.DB, planID)`.
       - Iterate operations in order. Skip any with status "completed". For each non-completed op:
         a. Check `ctx.Err()` — if cancelled, break loop, mark plan "failed".
         b. Set op status to "running" via `db.UpdateOperationStatus(e.DB, op.ID, "running", "")`.
         c. Call `e.executeOp(ctx, op)` to dispatch.
         d. If result.Success: call `db.UpdateOperationStatus(e.DB, op.ID, "completed", "")`.
         e. If !result.Success: call `db.UpdateOperationStatus(e.DB, op.ID, "failed", result.Error)`.
         f. Log audit entry via `db.LogAudit(e.DB, db.AuditEntry{...})` with EntityType "operation", EntityID as op.ID string, Action "execute", BeforeState JSON with source_path/dest_path, AfterState JSON with sha256/status/error, Success = result.Success, ErrorMessage = result.Error.
         g. Append result to results slice.
       - After loop: if any op failed or context cancelled, set plan to "failed" via `db.UpdatePlanStatusAudited`. Otherwise set to "completed".
       - Return results, nil.

    4. Implement `func (e *Executor) prepareResume(planID int64) error`:
       - Execute SQL: `UPDATE plan_operations SET status = 'pending', updated_at = CURRENT_TIMESTAMP WHERE plan_id = ? AND status = 'running'`

    5. Implement `func (e *Executor) executeOp(ctx context.Context, op db.PlanOperation) OpResult`:
       - Switch on `op.OpType`:
         - `"move"`: Call `fileops.VerifiedMove(op.SourcePath, op.DestPath)`. On success, call `fileops.HashFile(op.DestPath)` to get SHA-256.
         - `"flatten"`: Call `fileops.FlattenDir(op.SourcePath)`. If result has Errors, set error to first error's string. Collect SHA-256 from first FileMoveResult if available.
         - `"delete"`: Call `os.Remove(op.SourcePath)`.
         - `"write_metadata"`: Call `fileops.WriteMetadataSidecar(op.SourcePath, fileops.ABSMetadata{})` — note: for v1 this writes an empty/minimal sidecar; full metadata population comes in Phase 13+ when scan data is wired in.
         - default: Return OpResult with Error `fmt.Sprintf("unknown operation type: %s", op.OpType)`.
       - Return OpResult with OperationID, Success, SHA256, Error fields set.
