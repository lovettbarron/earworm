<objective>
Harden all file write paths with fsync, upgrade cross-filesystem moves to SHA-256 verification, and guard FlattenDir against cleanup when errors occurred.

Purpose: Prevent data loss on NAS mounts (NFS/SMB) where OS write caches can report success while data is still buffered. Size-only verification cannot detect bit-flip corruption. Directory cleanup after partial failures can orphan files.

Output: Safer copyFile, copyVerifyDelete, VerifiedCopy, and FlattenDir functions with targeted tests for each fix.
</objective>

```go
func hashFileSHA256(path string) (string, error) {
    f, err := os.Open(path)
    if err != nil { return "", err }
    defer f.Close()
    h := sha256.New()
    if _, err := io.Copy(h, f); err != nil { return "", err }
    return hex.EncodeToString(h.Sum(nil)), nil
}
```
Replace size check with: hash source before copy, hash destination after copy+sync+close, compare hashes, delete source only if match. Add `"crypto/sha256"` and `"encoding/hex"` to imports.
