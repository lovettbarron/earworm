# Quick Task 260405-m79: AAXC-to-M4B Decryption and Libation-Compatible Organization

**One-liner:** AAXC decryption via ffmpeg subprocess with voucher key/IV parsing, integrated into download pipeline between download and verify steps, with .m4b support in organize package.

## What Changed

### Task 1: Decrypt Module (internal/download/decrypt.go)

Created a complete AAXC decryption module:

- **ParseVoucher**: Reads audible-cli voucher JSON, extracts `content_license.license_response.key` and `iv`
- **DecryptAAXC**: Runs `ffmpeg -audible_key KEY -audible_iv IV -i input.aaxc -c copy output.m4b` via CmdFactory injection
- **FindAAXCAndVoucher**: Discovers .aaxc and .voucher files in a staging directory
- **DecryptStaged**: Orchestrates the full flow -- find, parse, decrypt, cleanup originals
- **CmdFactory type**: Allows test injection of mock ffmpeg (same pattern as audible package's cmdFactory)

12 unit tests covering: voucher parsing (valid, missing file, invalid JSON, missing key, missing IV), decrypt success/failure, file discovery, and full staged flow.

### Task 2: Pipeline and Organize Integration

**Pipeline (internal/download/pipeline.go):**
- Added `decryptFunc` field to Pipeline struct with test injection support
- `verifyAndMove` now takes context and calls `decryptFunc` before file verification
- Audio file glob expanded from `.m4a` only to `.m4b` then `.m4a` (prefers .m4b)
- Error message updated: "no audio files (.m4b/.m4a)" instead of "no M4A files"

**Organize (internal/organize/):**
- `destinationFilename` handles `.m4b` extension (renames to `Title.m4b`)
- `.aaxc` and `.voucher` files return empty string (skipped during organize)
- `OrganizeBook` skips files with empty destination names
- New `RenameAudioFile(title, ext)` generic function; `RenameM4AFile` delegates to it

**Tests:**
- Pipeline integration test for full AAXC download-decrypt-verify-move flow
- Organize tests for .m4b file handling and .aaxc/.voucher artifact skipping
- 3 new destinationFilename test cases (.m4b, .aaxc, .voucher)
