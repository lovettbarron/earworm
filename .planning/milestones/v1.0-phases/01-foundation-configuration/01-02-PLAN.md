**Step 2: Create `internal/config/paths.go`.**

Per D-04 and D-05, config lives at `~/.config/earworm/`:
```go
package config

import (
    "fmt"
    "os"
    "path/filepath"
)

// ConfigDir returns ~/.config/earworm/ (per D-04, hardcoded XDG path, NOT os.UserConfigDir which returns ~/Library/Application Support on macOS)
func ConfigDir() (string, error) {
    home, err := os.UserHomeDir()
    if err != nil {
        return "", fmt.Errorf("cannot determine home directory: %w", err)
    }
    return filepath.Join(home, ".config", "earworm"), nil
}

// ConfigFilePath returns ~/.config/earworm/config.yaml
func ConfigFilePath() (string, error) {
    dir, err := ConfigDir()
    if err != nil {
        return "", err
    }
    return filepath.Join(dir, "config.yaml"), nil
}

// DBPath returns ~/.config/earworm/earworm.db
func DBPath() (string, error) {
    dir, err := ConfigDir()
    if err != nil {
        return "", err
    }
    return filepath.Join(dir, "earworm.db"), nil
}
```

Functions:
- `SetDefaults()` -- sets all defaults above via `viper.SetDefault()`
- `InitConfig(cfgFile string) error` -- if cfgFile provided use it, otherwise look in ConfigDir() for config.yaml. Call SetDefaults() first, then viper.SetConfigType("yaml"), viper.AddConfigPath(configDir), viper.SetConfigName("config"). Call viper.ReadInConfig() -- ignore "not found" error (defaults are fine), return other errors.
- `Validate() error` -- check that library_path if non-empty points to an existing directory. Check that download.rate_limit_seconds > 0. Check that download.max_retries >= 0.
- `ValidKeys() []string` -- return sorted list of all valid config keys: `["audible_cli_path", "audiobookshelf.library_id", "audiobookshelf.token", "audiobookshelf.url", "download.backoff_multiplier", "download.max_retries", "download.rate_limit_seconds", "library_path", "staging_path"]`
- `WriteDefaultConfig(path string) error` -- creates the config directory (os.MkdirAll with 0755), writes a default config.yaml with comments explaining each key. Use viper.WriteConfigAs(path). Per Pitfall 3, always create directory first.
