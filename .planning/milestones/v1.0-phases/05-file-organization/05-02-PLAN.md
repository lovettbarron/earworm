    Define result types:
    ```go
    type OrganizeResult struct {
        ASIN      string
        Title     string
        Author    string
        LibPath   string // final library path
        Success   bool
        Error     string
    }
    ```

    Implement `OrganizeBook` function:
    ```go
    func OrganizeBook(book db.Book, stagingDir, libraryDir string) (string, error)
    ```
    Steps:
    1. Call `BuildBookPath(book.Author, book.Title, book.ASIN)` -- returns error if author/title missing per D-01
    2. Compute `destDir = filepath.Join(libraryDir, relPath)` and `srcDir = filepath.Join(stagingDir, book.ASIN)`
    3. `os.MkdirAll(destDir, 0755)` -- create Author/Title [ASIN] hierarchy. Per D-13, directory may already exist (overwrite)
    4. `os.ReadDir(srcDir)` -- list all files in staging ASIN dir
    5. For each file:
       - If extension is `.m4a`: destination filename = `RenameM4AFile(book.Title)` (per D-07)
       - If filename matches cover image (`.jpg`, `.jpeg`, `.png`): rename to `cover.jpg` (per D-05)
       - If filename matches chapter metadata (`.json`): rename to `chapters.json` (per D-06)
       - Otherwise: keep original filename
       - Call `MoveFile(srcFile, dstFile)` for each
    6. `os.Remove(srcDir)` -- remove empty staging dir (ignore error if not empty)
    7. Return destDir on success

    Implement `OrganizeAll` function:
    ```go
    func OrganizeAll(database *sql.DB, stagingDir, libraryDir string) ([]OrganizeResult, error)
    ```
    Steps:
    1. Call `db.ListOrganizable(database)` to get books with 'downloaded' status
    2. For each book:
       - Call `OrganizeBook(book, stagingDir, libraryDir)`
       - On success: call `db.UpdateOrganizeResult(database, book.ASIN, "organized", destDir, "")`
       - On error: call `db.UpdateOrganizeResult(database, book.ASIN, "error", "", err.Error())`
       - Append OrganizeResult to results slice
    3. Return all results (both successes and failures)

    **CLI Command (internal/cli/organize.go):**

    Create `earworm organize` command following established patterns:

    ```go
    package cli

    var (
        organizeJSON bool
    )

    var organizeCmd = &cobra.Command{
        Use:   "organize",
        Short: "Organize downloaded books into library folder structure",
        Long: `Move downloaded audiobooks from the staging directory into the library
    in Audiobookshelf-compatible Author/Title [ASIN]/ folder structure.

    Operates on all books with 'downloaded' status. Books missing required
    metadata (author, title) are marked as errors.`,
        RunE: runOrganize,
    }

    func init() {
        organizeCmd.Flags().BoolVar(&organizeJSON, "json", false, "output results in JSON format")
        rootCmd.AddCommand(organizeCmd)
    }
    ```

    `runOrganize` implementation:
    1. Get `staging_path` and `library_path` from viper -- validate both are non-empty, return clear error if not configured
    2. Open DB via `config.DBPath()` + `db.Open()`
    3. Call `organize.OrganizeAll(database, stagingPath, libraryPath)`
    4. Count successes and failures from results
    5. If `organizeJSON`:
       ```go
       type jsonOutput struct {
           Organized int              `json:"organized"`
           Errors    int              `json:"errors"`
           Results   []organize.OrganizeResult `json:"results"`
       }
       ```
       Encode and write to `cmd.OutOrStdout()`
    6. If not JSON and not `quietMode`:
       - Print each result: "Organized: Author - Title -> library/path" or "Error: Author - Title: error message"
       - Print summary: "Organized N books, N errors"
    7. If `quietMode` and not JSON: print only summary line
    8. Return nil (errors are per-book, not command-level -- the command itself succeeds even if individual books fail)
