### Integration Gaps (3)

5. **Cleanup from completed plans** (FOPS-03) — **DESIGN CONFLICT**
   Plan apply executes delete ops directly via `os.Remove` and marks them "completed". The cleanup command queries for pending delete ops from completed plans — after plan apply, none exist. The trash-dir safety mechanism (double confirm, reversible move) is unreachable in the normal workflow. The cleanup command works in isolation but no upstream workflow populates its queue.

6. **Scan→plan via Claude skill** (INTG-02) — **SKILL.md GAP**
   SKILL.md does not list `earworm scan issues`, `--create-plan`, or `earworm plan approve`. The scan→plan bridge workflow is absent from the skill. Core INTG-02 is satisfied (skill exists, covers split workflow), but scan-to-plan path requires CLI directly.

7. **Automated scan → write_metadata** (FOPS-02) — **BY DESIGN**
   The `missing_metadata` issue type is detected by scan but excluded from CreatePlanFromIssues (requires human review). Write_metadata ops only creatable via CSV import. FOPS-02 requirement is satisfied (the operation works), but there's no fully automated detection→plan path.
