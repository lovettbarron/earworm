Purpose: Users need CLI access to review and apply plans. This completes the scan-to-plan-to-apply workflow.

    **internal/cli/plan.go** — Create this file with package `cli`:

    1. Package-level flag variables:
       ```go
       var (
           planConfirm bool
           planJSON    bool
           planStatus  string
       )
       ```

    2. Parent command `planCmd`:
       - Use: "plan", Short: "Manage library cleanup plans"
       - No RunE (parent only)

    3. `planListCmd`:
       - Use: "list", Short: "List all plans", RunE: runPlanList
       - Flag: `--json` bound to `planJSON`, `--status` bound to `planStatus`
       - `runPlanList` opens DB via `config.DBPath()` + `db.Open()`, calls `db.ListPlans(database, planStatus)`.
       - If `planJSON`: marshal plans to JSON, print to cmd.OutOrStdout().
       - Else: print formatted table with columns: ID, Name, Status, Created. Use `fmt.Fprintf` with `%-6s %-30s %-12s %s\n` format. If empty, print "No plans found."

    4. `planReviewCmd`:
       - Use: "review [plan-id]", Short: "Review a plan's operations before applying", Args: cobra.ExactArgs(1), RunE: runPlanReview
       - Flag: `--json` bound to `planJSON`
       - `runPlanReview` parses plan ID from args[0] via `strconv.ParseInt`.
       - Opens DB, calls `db.GetPlan(database, planID)`. If nil, return `fmt.Errorf("plan %d not found", planID)`.
       - Calls `db.ListOperations(database, planID)`.
       - If `planJSON`: marshal struct `{Plan: plan, Operations: ops}` to JSON.
       - Else: print plan header (ID, Name, Status, operation count), then table with columns: `#` (Seq), Type, Status, Source, Destination using `fmt.Fprintf` with `%-4d %-16s %-10s %s\n` format. For ops with DestPath, show `source -> dest`. For ops without (delete), show just source.

    5. `planApplyCmd`:
       - Use: "apply [plan-id]", Short: "Apply a plan's operations", Args: cobra.ExactArgs(1), RunE: runPlanApply
       - Flag: `--confirm` bound to `planConfirm` (default false)
       - Flag: `--json` bound to `planJSON`
       - `runPlanApply` parses plan ID, opens DB.
       - If `!planConfirm`: call same display logic as review, then print "\nDry run — no changes made. Add --confirm to apply this plan.\n" and return nil.
       - If `planConfirm`:
         a. Create `executor := &planengine.Executor{DB: database}`
         b. Call `results, err := executor.Apply(cmd.Context(), planID)`
         c. Print per-operation results: seq number, pass/fail, SHA-256 if present, error if any
         d. Print summary: "Applied: X completed, Y failed out of Z operations"
         e. If `planJSON`: marshal results to JSON instead

    6. `init()` function:
       ```go
       func init() {
           planListCmd.Flags().BoolVar(&planJSON, "json", false, "output in JSON format")
           planListCmd.Flags().StringVar(&planStatus, "status", "", "filter by plan status")
           planReviewCmd.Flags().BoolVar(&planJSON, "json", false, "output in JSON format")
           planApplyCmd.Flags().BoolVar(&planConfirm, "confirm", false, "actually apply the plan (default is dry-run)")
           planApplyCmd.Flags().BoolVar(&planJSON, "json", false, "output in JSON format")
           planCmd.AddCommand(planListCmd, planReviewCmd, planApplyCmd)
           rootCmd.AddCommand(planCmd)
       }
       ```
