  <action>
    **grouper.go** -- Create `package split` with:

    Types:
    ```go
    type BookGroup struct {
        Title      string
        Author     string
        Narrator   string
        ASIN       string
        AudioFiles []string   // absolute paths
        Confidence float64    // 0.0-1.0
    }
    type GroupResult struct {
        SourceDir   string
        Groups      []BookGroup
        SharedFiles []string   // covers, metadata.json, etc.
        Skipped     bool
        SkipReason  string
    }
    ```

    `func GroupFiles(dirPath string) (*GroupResult, error)`:
    1. Call `metadata.FindAudioFiles(dirPath)` to get audio files.
    2. Read directory entries, collect non-audio files (extensions: .jpg, .jpeg, .png, .json) into SharedFiles list.
    3. Call `metadata.ExtractFileMetadata(filePath)` on EACH audio file individually.
    4. Group by normalized (lowercase) `title + "|" + author` key. Files with nil metadata go into "_unknown" group.
    5. If only 1 group (including _unknown merged in), return single group with Confidence=1.0 (not actually multi-book).
    6. If "_unknown" group has >20% of total files AND other groups exist, set Skipped=true, SkipReason="too many files without metadata for confident grouping".
    7. Calculate per-group confidence: (files with matching title+author) / (total files in group). Overall confidence = min of all group confidences.
    8. If overall confidence < 0.7, set Skipped=true, SkipReason="confidence below threshold: {confidence}" (per D-03).
    9. For filename fallback: if ALL files have nil metadata, attempt filename pattern grouping using `extractTitle` style logic (look for common prefixes, book title patterns like "Book1_Chapter01.m4a"). If still ambiguous, skip.
    10. Inject `extractFileMetadataFn` as package-level var for test seam: `var extractFileMetadataFn = metadata.ExtractFileMetadata`.

    **planner.go** -- Create:

    `func CreateSplitPlan(database *sql.DB, result *GroupResult, libraryRoot string) (int64, error)`:
    1. If result.Skipped, return 0 and error "cannot create plan for skipped folder".
    2. Create plan via `db.CreatePlan(database, "split: "+filepath.Base(result.SourceDir), "Split multi-book folder into individual book directories")`.
    3. seq counter starting at 1.
    4. For each BookGroup:
       a. Call `organize.BuildBookPath(group.Author, group.Title, group.ASIN)` for relative dest dir. If author/title empty, use "Unknown Author"/"Unknown Title".
       b. Full dest dir = `filepath.Join(libraryRoot, relPath)`.
       c. For each audio file: `db.AddOperation(database, db.PlanOperation{PlanID: planID, Seq: seq, OpType: "split", SourcePath: audioFile, DestPath: filepath.Join(fullDestDir, filepath.Base(audioFile))})`. Increment seq.
       d. For each shared file in result.SharedFiles: `db.AddOperation(database, db.PlanOperation{PlanID: planID, Seq: seq, OpType: "split", SourcePath: sharedFile, DestPath: filepath.Join(fullDestDir, filepath.Base(sharedFile))})`. Increment seq.
    5. Return planID.

    Shared files appear once per group in operations, each with a different DestPath. The plan engine (Plan 02) distinguishes copy vs move by file extension at execution time.
  </action>
