## Tech Debt Summary

### Dead API Surface (2 items)

1. **`db.InsertBook`** (internal/db/books.go) — defined but never called in production. All insertion paths use `UpsertBook` or `SyncRemoteBook`. Should be unexported or removed.

2. **`organize.RenameM4AFile`** (internal/organize/path.go) — exported but only called within the package. Should be unexported or removed.

### Redundant Behavior (1 item)

3. **Double ABS scan in daemon cycle** — `cli/daemon.go` explicitly calls `abs.ScanLibrary()` AND `runOrganize` (called just before) already triggers ABS scan inside `cli/organize.go`. When books are organized, ABS receives two sequential POST scan requests per cycle. Benign but wasteful.

### Documentation Completeness (1 item)

4. **SUMMARY.md frontmatter gaps** — The `requirements_completed` field is not populated for ~50% of plan summaries. All requirements are verified via VERIFICATION.md, so this is a documentation completeness issue only.

**Total: 4 items across 4 phases**
